CP3403 - Data Mining Assignment 1

Gender Recognition by Voice Analysis — A comparative study of Decision Tree and Naive Bayes classifiers on 44 acoustic features.

Project Overview

Voice recognition is increasingly used in authentication, smart devices, and personalised user experiences. This project analyses 44 vocal features — including spectral centroids, MFCCs, pitch, and energy measures — to classify a speaker’s gender (male / female). The dataset (16,148 rows) was preprocessed via normalisation and duplicate removal (1,078 duplicates eliminated).

We built and compared two classifiers: a Decision Tree (with entropy splitting) and a Naive Bayes (GaussianNB) model. Additionally, we applied Association Rule Mining (ARM) using the Apriori algorithm to discover interpretable patterns between binned audio features and gender.

Key Results

  • Naive Bayes (GaussianNB) — 93% accuracy, with balanced recall:
    • Female recall: 0.95
    • Male precision: 0.97
  • Decision Tree — 90% accuracy, but lower recall for females (0.77) due to class imbalance.

The Naive Bayes model proved more resilient to the dataset’s imbalance (~50% more male samples) and is recommended for real‑world deployment due to its simplicity, speed, and strong performance.

Association Rule Mining (ARM)

Using the Apriori algorithm on binned features (Low/Medium/High), we discovered meaningful rules — for example, “when mean_spectral_contrast is Low, there is a high likelihood the voice belongs to a Male.” These rules provide interpretable insights that complement the black‑box models.
